* Why do moder-append and moder-insert behave the same way when there is no region?

In Emacs cursor is located between two characters, so we would have to implement a hack to change the behaviour. You can read more about it in [[https://github.com/moder-edit/moder/discussions/87][this discussion thread]].

* How to customize the +prefix menus in Keypad mode?

Please check [[https://github.com/moder-edit/moder/issues/71#issuecomment-962090002][This issue]].

* I encountered a weird command behavior in normal state

Moder's internal command will not call Emacs' commands directly. They will call a keybind which is then linked to a command.  Your customized keybind might be overriding the original command. Lookup the command you want to execute ([[moder-var.el][moder-var.el]]) and overwrite it in your config.

[[https://github.com/moder-edit/moder/issues/109][Related issue]]

Example:

Here you can map Delete character to internal Moder's delete character function.
#+begin_src emacs-lisp
  (setq moder--kbd-delete-char "<deletechar>")
#+end_src

The keyboard macro variables can also be assigned to defuns if you don't want them to correspond to a keyboard macro.

Example:
#+BEGIN_SRC emacs-lisp
  (setq moder--kbd-forward-line #'next-line)
#+END_SRC

* Why don't the number hints for extending selection show up in org-mode buffers?

The extend selection number hints are disabled in org-mode by default, because they work poorly with variable-width fonts, and fonts with different sizes. If you use a fixed width font and a single font size, the number hints can be enabled in org mode by removing the org symbol from the list moder-expand-exclude-mode-list . See [[https://github.com/moder-edit/moder/issues/110][this issue for more details]].
